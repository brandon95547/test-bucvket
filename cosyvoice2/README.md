# CosyVoice 2 — voice clone test

Zero-shot voice clone with [CosyVoice 2](https://github.com/FunAudioLLM/CosyVoice)
(CosyVoice2-0.5B). Clones the shared `../reference/strong_ref.wav` and reads the shared
`../test.txt` in that voice → `outputs/strong_clone.wav` (24 kHz).

CosyVoice needs **Python 3.10**, a repo clone (+ Matcha-TTS submodule), and a ~2 GB model
download — so the setup is scripted rather than a plain `pip install`.

## Setup (one-time)
```bash
cd cosyvoice2
./setup.sh
```
This clones CosyVoice, makes `.venv` (Python 3.10), installs deps, and downloads the model.
Prereqs: `python3.10` (or `uv`), plus system `sox` (`dnf install sox sox-devel`) and `ffmpeg`.

## Run
```bash
.venv/bin/python clone_test.py
```
Uses the GPU automatically if one is visible. Output: `outputs/strong_clone.wav`.

## Notes
- **Reference:** shared with the other engines (`../reference/strong_ref.wav` + `.txt`).
  CosyVoice conditions on the transcript, so `strong_ref.txt` should match the clip.
  Prompt clip must be ≤ 30 s (3–10 s ideal).
- Input is split into `COSY_MAX_CHARS` (default 300) sentence chunks and concatenated.
- Everything heavy is gitignored: `.venv/`, the cloned `CosyVoice/` repo, and the model
  under `CosyVoice/pretrained_models/` — all regenerated by `setup.sh`.
- API note: this targets current CosyVoice `main`, where `inference_zero_shot` takes the
  prompt **wav path** (not a preloaded tensor).
